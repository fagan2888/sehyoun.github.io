Writing the setup_eqns Function¶

For the people from DYNARE/RISE tradition(?), this function can be considered as the “model” section of DYNARE/RISE model files. Unlike DYNARE/RISE, there is no parsing at this stage of the program, so one needs to write MATLAB functions with the restriction that it has to be supported by MATLABAutoDiff . However, that’s the only restriction, and one can write any sort of expressions.
