Sparse Grid Finite Difference Codes¶

There are already open-source implementation of sparse-grid interpolation toolbox (refer to SPGrid++. However, I found them to have too many features. There is always a trade-off between availability of features and complexity of the code-base. For my application, I decided that the payoff is in the negative, and hence, I have implemented things myself. I might introduce more features in the future, but currently, there are just enough codes to implement finite difference methods based on upwind scheme.

Specifications:¶

Estimated time to use for basic usage:

  • 5 minutes

sparse_grid.compute_derivs(func, dimension)¶

INTERNAL FUNCTION: Compute forward and backward derivative values for a given function

Parameters:
  • func (vector) – vector/matrix of values of the function to take derivative of
  • dimension (int) – coordinate dimension to take derivative in
  • d_forward (implicit) – d_forward
  • d_backward (implicit) – d_backward
Returns:

[dv_dxf, dv_dxb]

  • dv_dxf: forward derivative
  • dv_dxb: backward derivative

Warning

In most regular applications, one can call build_upwind, and hence do not need to consider derivative values.

sparse_grid.find_upwind(mu_f, mu_b, dimension)¶

INTERNAL FUNCTION: Findings direction for upwind

Parameters:
  • mu_f (vector) – (n_points x 1) vector of “forward” drifts
  • mu_b (vector) – (n_points x 1) vector of “backward” drifts
  • dimension (int) – coordinate dimension the drifts are defined
Returns:

[ind_f, ind_b]

  • ind_f : (n_points x 1) vector whether forward direction should be used
  • ind_b : (n_points x 1) vector whether backward direciton should be used

Note

  • For \(\mu\) that’s not defined separately, you can just call find_upwind(mu,mu,dimension)
  • One probably want to call methods directly in MFG class instead of working with grids directly.
