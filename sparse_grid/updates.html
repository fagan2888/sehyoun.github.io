Potential UpdatesÂ¶

This toolbox is written mainly for personal and work related use at Norges Bank. Hence, things are written for specific applications. General extensions are left for future if the expected payoff for the current application is small. Create an issue in github if you believe some extension might be useful for you application.

Todo

  • Separate out recompute_grid to allow computing only the required operators.
  • Create an option to treat the boundary condition differently for dd_center. Currently, it assumes that the boundary is a reflecting boundary.
  • Only multithreading is supported via openmp. In theory, the problem can be extensioned with MPI to use even more compute nodes. Currently, the internal problem size is not large enough to require MPI setup.
  • Adaptation is done in an (asymptotical) inefficient manner. As the scale of the problem increases, this part can take a larger share of the program execution, but currently, adaptation takes less than a fraction of one percent of the execution time.
